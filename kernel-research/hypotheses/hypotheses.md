# Kernel Optimization Hypotheses
Generated by serious-inference-engineer coordination branch + ooo collaboration
Last updated: 2026-03-23

## Schema
Each hypothesis:
- hypothesis_id: h001, h002, ...
- kernel_type: attention | GEMM | fused-ops | other
- method: concrete description
- measurable_criteria: what metric proves it
- estimated_difficulty: low | medium | high
- priority_rank: 1 = highest priority

## Categories
- [BW] Memory bandwidth reduction
- [CR] Compute reuse
- [FO] Fused operator patterns
- [QU] Quantization-aware
- [SC] Scheduling/parallelism

## Ranked Hypotheses

### h001 -- FP8 End-to-End Attention Pipeline [BW, QU]
- **kernel_type**: attention
- **method**: Cast Q/K/V to FP8 (e4m3) before QK^T matmul, accumulate in FP16, return FP16 output. Uses torch._scaled_mm on H100 SM90.
- **measurable_criteria**: speedup >= 1.10x on H100, max_abs_err < 1e-4 vs FP16 reference, batch [1,8,32]
- **estimated_difficulty**: medium
- **priority_rank**: 1
- **expected_speedup**: 1.3-1.5x (FP8 throughput 2x vs FP16 on H100)
- **notes**: Requires CUDA >= 12.0, PyTorch >= 2.1, H100 GPU. Primary target for sprint.

### h002 -- Fused RMSNorm + RoPE + QKV Projection [FO]
- **kernel_type**: fused-ops
- **method**: Single Triton kernel that computes RMSNorm(x), applies RoPE positional encoding, and projects to QKV. Eliminates 2 global memory round-trips.
- **measurable_criteria**: speedup >= 1.10x vs separate ops, correctness vs layerwise reference
- **estimated_difficulty**: high
- **priority_rank**: 2
- **expected_speedup**: 1.2-1.4x (eliminates 2 DRAM reads, 2 DRAM writes)
- **notes**: Key bottleneck in decoder-only LLM inference. Critical for autoregressive decode.

### h003 -- W4A8 SplitK GEMM [QU, CR]
- **kernel_type**: GEMM
- **method**: Weight-only INT4 with FP8 activations (W4A8). Use SplitK algorithm to split K-dimension across thread blocks, accumulate in FP32. Dequant weight on-the-fly from INT4 to FP8.
- **measurable_criteria**: speedup >= 1.10x vs cuBLAS FP16 GEMM, max_abs_err < 1e-3 (relaxed for INT4 weights)
- **estimated_difficulty**: high
- **priority_rank**: 3
- **expected_speedup**: 1.5-2.0x (bandwidth bound; 4-bit weights = 4x bandwidth reduction)
- **notes**: Best for decode-phase GEMMs where M=1 (batch=1 decode). Memory-bandwidth dominant.

### h004 -- CUDA Graphs for Decode Loop [SC]
- **kernel_type**: fused-ops
- **method**: Capture the full decode step (attention + MLP + sampling) as a CUDA Graph. Eliminates kernel launch overhead and CPU-GPU synchronization in the critical path.
- **measurable_criteria**: speedup >= 1.10x for decode latency, variance < 1% (graphs eliminate jitter)
- **estimated_difficulty**: medium
- **priority_rank**: 4
- **expected_speedup**: 1.15-1.30x for batch=1 decode (launch overhead dominant)
- **notes**: Highest impact for latency-critical single-request decode. Fully orthogonal to other methods.

### h005 -- FlashAttention-3 FP8 with MLA [BW, QU]
- **kernel_type**: attention
- **method**: Flash-attention-3 kernel modified for Multi-head Latent Attention (DeepSeek MLA): separate KV compression, FP8 softmax, SM90 wgmma instruction.
- **measurable_criteria**: speedup >= 1.20x vs FA2 baseline, correctness < 1e-4
- **estimated_difficulty**: high
- **priority_rank**: 5
- **expected_speedup**: 1.3-2.0x (FA3 + FP8 combined on H100)
- **notes**: Requires custom Triton or CUDA kernel. Highest ceiling but hardest to implement.

### h006 -- Blocked FP16 GEMM with Shared Memory Tiling [CR, BW]
- **kernel_type**: GEMM
- **method**: Custom Triton GEMM with explicit 128x128 tile sizes optimized for H100 shared memory (228KB). Double-buffered async memory loads. Compare against cuBLAS.
- **measurable_criteria**: speedup >= 1.10x vs torch.matmul for shapes [512x512, 1024x1024, 4096x4096]
- **estimated_difficulty**: medium
- **priority_rank**: 6
- **expected_speedup**: 1.0-1.2x (cuBLAS is near-optimal; headroom mostly for specific shapes)
- **notes**: Good starting point for CPU benchmarking to validate correctness before GPU runs.

### h007 -- Fused LayerNorm + Linear [FO]
- **kernel_type**: fused-ops
- **method**: Single Triton kernel: compute LayerNorm and immediately project through a linear layer. Avoids materializing the normalized activations to DRAM.
- **measurable_criteria**: speedup >= 1.10x vs F.layer_norm + torch.matmul, max_abs_err < 1e-4
- **estimated_difficulty**: low
- **priority_rank**: 7
- **expected_speedup**: 1.1-1.3x
- **notes**: Well-studied optimization. CPU-benchmarkable for correctness.

### h008 -- Persistent Kernel for Decode Attention [SC, CR]
- **kernel_type**: attention
- **method**: Persistent CUDA kernel that keeps thread blocks alive across multiple decode steps. Eliminates kernel launch overhead per token. Thread blocks process a fixed head-of-work queue.
- **measurable_criteria**: speedup >= 1.10x for decode loop over 128 tokens, variance < 2%
- **estimated_difficulty**: high
- **priority_rank**: 8
- **expected_speedup**: 1.15-1.25x

### h009 -- INT8 SmoothQuant Attention [QU]
- **kernel_type**: attention
- **method**: SmoothQuant migration: migrate quantization difficulty from activations to weights. Both W and X in INT8. Use channel-wise scale migration factors.
- **measurable_criteria**: speedup >= 1.10x, max_abs_err < 1e-3 (INT8 error budget)
- **estimated_difficulty**: medium
- **priority_rank**: 9
- **expected_speedup**: 1.2-1.6x

### h010 -- Chunked Prefill with Fused KV Cache Append [FO, BW]
- **kernel_type**: attention
- **method**: Process prefill in chunks of 512 tokens. For each chunk, fuse the KV-cache write with the attention computation to avoid materializing full K/V matrices.
- **measurable_criteria**: speedup >= 1.10x for seq_len=2048, memory usage reduction
- **estimated_difficulty**: medium
- **priority_rank**: 10

### h011 -- Warp-Specialized GEMM for Small M [SC, CR]
- **kernel_type**: GEMM
- **method**: For decode-phase GEMM (M=1-8), use warp-specialized split-K: one warp group loads tiles, another computes. Reduces latency vs monolithic kernel.
- **measurable_criteria**: speedup >= 1.10x for M in [1,4,8], K=4096, N=4096
- **estimated_difficulty**: medium
- **priority_rank**: 11

### h012 -- Fused GELU + Linear [FO]
- **kernel_type**: fused-ops
- **method**: Fuse GELU activation with subsequent linear projection. Eliminates intermediate activation materialization. Use fast GELU approximation (tanh-based).
- **measurable_criteria**: speedup >= 1.10x vs F.gelu + torch.matmul
- **estimated_difficulty**: low
- **priority_rank**: 12
- **expected_speedup**: 1.1-1.2x
- **notes**: CPU-benchmarkable.

### h013 -- Paged KV Cache with Memory Pool [BW]
- **kernel_type**: attention
- **method**: Block-based KV cache (page size 16 tokens). Allows non-contiguous KV storage. Eliminates cache copy on context extension. Uses CUDA memory pool for zero-overhead allocation.
- **measurable_criteria**: reduction in peak memory, speedup >= 1.05x for variable-length batches
- **estimated_difficulty**: high
- **priority_rank**: 13

### h014 -- Softmax Temperature Fusion [FO]
- **kernel_type**: attention
- **method**: Fuse temperature scaling (logits / temp) into the softmax kernel. Avoid materializing temperature-scaled logits.
- **measurable_criteria**: speedup >= 1.05x vs separate ops
- **estimated_difficulty**: low
- **priority_rank**: 14

### h015 -- Vectorized Token Embedding Lookup [BW]
- **kernel_type**: other
- **method**: Vectorized gather for token embedding lookup using 128-bit loads (float4). Coalesced global memory access pattern for batch token embedding.
- **measurable_criteria**: speedup >= 1.10x for vocab_size=32000, batch=[1,8,32]
- **estimated_difficulty**: low
- **priority_rank**: 15

### h016 -- Fused RoPE with Flash Attention [FO, BW]
- **kernel_type**: attention
- **method**: Apply RoPE positional encoding inside the flash attention kernel. Avoids materializing RoPE-applied Q/K before attention.
- **measurable_criteria**: speedup >= 1.10x vs RoPE + FA2, correctness < 1e-4
- **estimated_difficulty**: medium
- **priority_rank**: 16

### h017 -- L1-Cache Optimized Softmax [CR, BW]
- **kernel_type**: fused-ops
- **method**: Online softmax (flash-softmax) implementation that processes sequence in a single pass. L1-cache-friendly tiling for seq_len <= L1_cache / (4*bytes_per_score).
- **measurable_criteria**: speedup >= 1.10x for seq_len=[512, 1024, 2048]
- **estimated_difficulty**: medium
- **priority_rank**: 17

### h018 -- Async Prefetch for KV Cache [BW, SC]
- **kernel_type**: attention
- **method**: Use CUDA async copy (cp.async) to overlap KV cache loading with QK^T computation. Tensor Memory Accelerator (TMA) on H100 for bulk async transfers.
- **measurable_criteria**: speedup >= 1.10x for long-context (seq_len=4096+), memory bandwidth utilization > 80%
- **estimated_difficulty**: high
- **priority_rank**: 18

### h019 -- Batched GEMM with Custom Stream Parallelism [SC]
- **kernel_type**: GEMM
- **method**: Split batched GEMM across multiple CUDA streams with explicit synchronization. Better for irregular batch sizes where standard batched GEMM underutilizes SMs.
- **measurable_criteria**: speedup >= 1.10x for irregular batch sizes [1,3,7,13]
- **estimated_difficulty**: medium
- **priority_rank**: 19

### h020 -- Speculative GEMM with Early Exit [CR]
- **kernel_type**: GEMM
- **method**: For attention score GEMMs, use sparse row pattern from speculative mask to skip zero regions early. Works with sparse attention patterns.
- **measurable_criteria**: speedup >= 1.10x for sparsity >= 30%, correctness < 1e-4
- **estimated_difficulty**: high
- **priority_rank**: 20
